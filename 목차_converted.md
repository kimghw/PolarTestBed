##### 참고: 삭제/통합된 항목

### (가상환경) 섹션에서 통합된 항목:
- 시뮬레이션 기반 위험예측 → 2번 실시간 시뮬레이션 플랫폼에 통합
- 극지환경 가상 모델링 구축 → 1번 디지털트윈 모델링에 통합
- 주요 추진설비 디지털트윈 모델 → 1번 디지털트윈 모델링에 통합
- 실시간 모니터링 및 대시보드 → 3번 디지털트윈 통합 플랫폼에 통합
- 데이터 저장 및 분석 → 3번 디지털트윈 통합 플랫폼에 통합
- 디지털트윈 V&V 운영 절차 → (디지털시험) 섹션 2번으로 이동/확장

### (시험기술) 섹션에서 통합된 항목:
- 하이브리드 시스템 제어 통합시험 기술(HIL/PHIL) → 2번으로 간소화
- 통신 인터페이스 시험 → 3번 시스템 인터페이스 및 안전성 시험에 통합
- 비상운용 시험(비상정지 시스템 검증) → 3번 시스템 인터페이스 및 안전성 시험에 통합

### (지능형) 섹션에서 통합된 항목:
- LLM을 활용한 시험모델 생성 → 1번으로 통합
- LLM을 활용한 시험 시나리오 자동 생성 → 1번으로 통합
- LLM을 활용한 센서 데이터 매핑 시스템 → 2번으로 통합
- 딥러닝 기술을 활용한 이상탐지 → 2번으로 통합
- 시험결과 자동 분석 및 리포팅 템플릿 자동화 → 3번으로 통합
- LLM 사용 범위/휴먼인루프 추적 → 삭제 (AI 시스템 자체 관리는 시험 지원과 직접적 관련성 낮음)
- 도메인 특화 LLM 파인튜닝 (극지/선박 도메인 데이터, 전문 용어 학습, 시험 프로토콜 특화 모델) → 4번 RAG 시스템의 세부 항목으로 통합

### 비용 관련 항목:
- (비용) 데이터 인프라 → (인프라) 4번으로 명시
- (비용) 실시간 시뮬레이션 플랫폼 → (가상환경) 2번에 포함
- (옵션) 클라우드 시스템 → (인프라) 4번 데이터 인프라에 포함 가능

### 제거된 항목:
- ESS 시험동 → 부하 설비에 통합 (1번 "계통 or ESS 등")
- 정밀 계측 및 센서 시스템 → 3번 추진시스템 전용 설비와 통합
- (인프라) 시설 운영 및 유지보수 → (시험설비) 4번으로 이동
